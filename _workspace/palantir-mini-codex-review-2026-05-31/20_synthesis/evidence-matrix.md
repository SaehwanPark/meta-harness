## Synthesis Notes

- Report 02 documents the old/bottleneck behavior observed during this review. Reports 07 and 09, plus the targeted source read in this synthesis, document the newer checked-in source state where Codex `PreToolUse` is intentionally absent. The final proposal treats both as true at different times.
- The Lead hotfix is source-level evidence, not active-session evidence. `docs/RELOAD_PER_RUNTIME.md` states that runtime-surface changes require plugin reinstall/reload and Codex restart before the active session sees them.
- The proposal should reduce exposure first and delete files last. File existence is less dangerous than default public availability, broad hook attachment, mixed runtime authority, and false parity claims.
